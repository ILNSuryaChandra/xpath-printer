## Codebase Patterns

- Libraries (html2canvas, jsPDF) are bundled locally in `libs/` and injected via `chrome.scripting.executeScript` in the background service worker **before** messaging the content script.
- Content script is registered via `manifest.json` `content_scripts` array (not injected programmatically), so it is always present and ready to receive messages.
- The popup sends a message to the background worker with the tab ID; it does NOT talk to the content script directly.

## What was implemented

SNAP-005 (Rendering Engine — DOM-to-Canvas via html2canvas):

- Added `renderedCanvas` global state to hold the html2canvas output for the PDF pipeline.
- `renderSelectedElement()` — async function that:
  1. Shows a full-screen processing overlay with animated spinner and "Rendering element…" label (backdrop-filter blur).
  2. Calls `neutralizePrintStyles()` to temporarily delete all `@media print` CSS rules from stylesheets, ensuring html2canvas sees only screen styles.
  3. Invokes `html2canvas(selectedElement, { scale:2, useCORS:true, allowTaint:false, removeContainer:true, backgroundColor:null })` for high-DPI, cross-origin-safe rendering.
  4. Restores print styles in a `finally` block (always runs even on error).
  5. On success: removes processing overlay, shows green success toast, stores canvas in `renderedCanvas`.
  6. On failure: removes processing overlay, shows red error toast with message, clears `renderedCanvas`.
- `neutralizePrintStyles()` — iterates all document stylesheets, finds `@media print` rules, deletes them (reverse order to preserve indices), and returns a `restore()` closure that re-inserts them.
- `createProcessingOverlay()` / `removeProcessingOverlay()` — full-screen dark overlay with CSS spinner animation.
- `showToast(msg, isError)` — bottom-center toast notification with auto-fadeout after 2.8s and removal after 3.4s.
- `exitSelectionMode(false)` now calls `renderSelectedElement()` instead of the previous TODO placeholder.

## Files changed

content.js, prd.json (SNAP-005 passes: true)

## Learnings for future iterations

Patterns discovered:
- html2canvas is injected via `chrome.scripting.executeScript` in the MAIN world (background.js) before the content script runs, so `html2canvas` is available as a global in the page context. Content scripts run in an isolated world by default — if html2canvas isn't visible, it means the lib injection and content script are in different worlds. Current setup works because content.js is registered in manifest.json and runs in the page's main world context alongside the injected libs.
- `neutralizePrintStyles()` must delete rules in reverse index order — otherwise deleting rule at index 2 shifts rule at index 3 to index 2, causing off-by-one errors.
- `backgroundColor: null` in html2canvas preserves the element's actual background (including transparency), rather than forcing a white background.

Gotchas:
- Cross-origin stylesheets throw a SecurityError when accessing `sheet.cssRules` — the `try/catch` in `neutralizePrintStyles` is essential.
- html2canvas does not support all CSS features (e.g., CSS Grid, some flexbox edge cases, backdrop-filter). This is a known limitation — the output is "good enough" for most real-world captures.
- The processing overlay must be cleaned up in both success and error paths — using try/catch ensures no phantom overlays remain on the page.

## Learnings for future iterations

Patterns discovered:
- jsPDF UMD exposes itself as `window.jspdf.jsPDF`.
- To split a single tall image across multiple PDF pages, you re-draw the same image on each new page but shift its negative Y-coordinate up by the total height already rendered.

Standard slicing involves drawing the image at `y = -offset`.
